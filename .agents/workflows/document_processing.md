# Навык: Обработка Документов

Этот навык позволяет агенту Antigravity сканировать и обрабатывать документы различных форматов, таких как DOCX, PDF, TXT и другие, для извлечения текстового содержимого и метаданных. Полученные данные будут использоваться для дальнейшего анализа или интеграции в проект пользователя.

## Когда использовать этот навык

Используйте этот навык, когда необходимо:

*   Извлечь текстовое содержимое из файлов DOCX, PDF или других текстовых документов.
*   Получить метаданные документов (например, автор, дата создания, количество страниц).
*   Подготовить текстовые данные для индексации, поиска или анализа.
*   Обработать большое количество документов в автоматическом режиме.

## Как использовать этот навык

Агент должен следовать следующим шагам для эффективного сканирования и обработки документов:

### 1. Идентификация документов

1.  **Определение местоположения:** Агент должен определить директорию или список файлов, которые необходимо обработать. Предполагается, что документы находятся в рабочей директории проекта или в указанном пользователем пути.
2.  **Распознавание формата:** Для каждого файла агент должен определить его формат (например, `.docx`, `.pdf`, `.txt`). Это можно сделать с помощью расширения файла или анализа его содержимого.

### 2. Использование MCP серверов для обработки

Для обработки документов агент должен использовать соответствующие MCP (Model Context Protocol) серверы, если они доступны и настроены. Это обеспечивает эффективное и надежное извлечение данных.

*   **Для PDF-файлов:** Используйте `pdf-reader-mcp` или аналогичный MCP сервер для извлечения текста и метаданных. Пример команды (если MCP сервер предоставляет CLI): 
    `manus-mcp-cli pdf-reader-mcp read --file <path_to_pdf> --output <output_file.txt>`.
*   **Для DOCX-файлов:** Используйте `docx-processor-mcp` или аналогичный MCP сервер для извлечения текста и метаданных. Пример команды: 
    `manus-mcp-cli docx-processor-mcp extract-text --file <path_to_docx> --output <output_file.txt>`.
*   **Для TXT-файлов:** Просто прочитайте содержимое файла напрямую.
*   **Для других форматов:** Если доступны другие MCP серверы или инструменты командной строки, используйте их. В противном случае, агент должен уведомить пользователя о невозможности обработки данного формата.

### 3. Извлечение и сохранение содержимого

1.  **Извлечение текста:** Извлеките весь доступный текст из документа.
2.  **Извлечение метаданных:** По возможности извлеките ключевые метаданные (например, название, автор, дата, количество страниц).
3.  **Сохранение результатов:** Сохраните извлеченный текст в отдельный текстовый файл (например, `<original_filename>.txt`) в поддиректории `processed_documents` в рабочей директории. Метаданные могут быть сохранены в отдельном JSON-файле или добавлены в начало текстового файла.

### 4. Обработка ошибок

*   Если документ не может быть открыт или обработан, агент должен записать ошибку в лог-файл и перейти к следующему документу.
*   Уведомить пользователя о любых критических ошибках или необработанных файлах.

## Ожидаемый результат

После выполнения этого навыка в директории `processed_documents` должны находиться текстовые файлы, содержащие извлеченное содержимое из каждого обработанного документа, а также, при необходимости, файлы с метаданными или лог-файл с информацией об ошибках.

## Лучшие практики

*   **Пакетная обработка:** При работе с большим количеством файлов используйте пакетную обработку для повышения эффективности.
*   **Логирование:** Ведите подробный лог всех обработанных файлов, извлеченных данных и возникших ошибок.
*   **Конфигурируемость:** Если возможно, позвольте пользователю указывать входную директорию, выходную директорию и типы файлов для обработки.
*   **Безопасность:** Убедитесь, что обработка документов не приводит к утечке конфиденциальной информации или выполнению вредоносного кода.
