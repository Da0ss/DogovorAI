# Правила разработки (Development Skills)

### 1. Качество кода
- **Clean Code:** Придерживаться принципов SOLID и DRY.
- **Типизация:** Обязательное использование Type Hints в Python и строгая типизация в TypeScript.
- **Асинхронность:** Весь I/O (запросы к API, работа с базой) должен быть `async/await`.

### 2. Структура проекта
Соблюдать модульную архитектуру:
- `/app/api` — эндпоинты
- `/app/services` — бизнес-логика (AI, OCR, Legal Logic)
- `/app/models` — схемы данных (Pydantic/Supabase)
- `/tests` — тесты для каждого модуля

### 3. Тестирование
- Каждая новая фича должна сопровождаться Unit-тестами (Pytest).
- Покрытие тестами критических узлов (анализ рисков) не менее 80%.

### 4. Документация 
- Все функции должны иметь Google-style docstrings.
- Язык кода: Английский (переменные, функции).
- Язык комментариев и логов: Русский (для удобства отладки).
